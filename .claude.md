# Retrieva - RAG Platform

## Architecture
- **Backend**: Python 3.11+ / FastAPI (api/)
- **Core Engine**: Ingestion, Retrieval, Generation (core/)
- **Workers**: Celery + Redis (workers/)
- **Frontend**: Next.js + TypeScript + Tailwind (dashboard/)
- **Vector DB**: Qdrant
- **Database**: PostgreSQL + SQLAlchemy async
- **Queue/Cache**: Redis + Celery

## Conventions
- Python: PEP 8, type hints everywhere, async/await for I/O
- API routes under /api/v1/
- Pydantic for all request/response schemas (api/schemas/)
- SQLAlchemy models in api/models/
- All core engine code in core/ (no API dependencies)
- Connectors implement BaseConnector interface

## Rules
- Never hardcode API keys or secrets - use .env
- All database access via async sessions
- Workers use sync DB sessions (Celery limitation)
- Config loaded from config.yaml, overridable via env vars
- Multi-tenant: always scope queries by tenant_id
